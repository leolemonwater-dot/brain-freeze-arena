# 作業承認フロー

ユーザーから作業を依頼された場合、Kiroは以下のフローに従ってください。

## 基本ルール

1. **作業前に計画を説明する**
   - 何をどのように変更・実装するかを箇条書きで簡潔に説明する
   - ユーザーの承認を得てから作業を開始する
   - 承認前にファイルの編集やコマンドの実行は行わない

2. **作業量が多い場合は分割して進める**
   - 作業ステップが3つ以上、または複数ファイルにまたがる場合は分割を検討する
   - 各フェーズの完了後にユーザーに確認を取り、次のフェーズに進む
   - 例：「フェーズ1が完了しました。次はフェーズ2（〇〇）に進んでよいですか？」

## 計画説明の形式

```
## 作業計画

**目的:** （何を達成するか）

**変更内容:**
1. 〇〇を修正する
2. 〇〇を追加する
3. 〇〇をテストする

**影響範囲:** （変更されるファイルや機能）

進めてよいですか？
```

## 例外

- 単純な質問への回答（コードを変更しない場合）は承認不要
- ユーザーが「すぐにやって」「承認不要」と明示した場合はそのまま進める
